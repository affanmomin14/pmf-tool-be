**CRITICAL: ProblemType-to-slug mapping.** The ProblemType enum values use underscores but category slugs use hyphens, AND there is a semantic mismatch: `product_quality` maps to slug `positioning` (not `product-quality`). Create an explicit mapping:

```typescript
const PROBLEM_TYPE_TO_SLUG: Record<string, string> = {
  market_fit: 'market-fit',
  product_quality: 'positioning',
  distribution: 'distribution',
  monetization: 'monetization',
  retention: 'retention',
};
```

**Function 1: `createAssessment`**
- Accepts: `{ problemType: ProblemType, utmSource?: string, utmMedium?: string, utmCampaign?: string, ipHash: string }`
- Uses `prisma.$transaction(async (tx) => { ... })` for atomicity
- Inside transaction: (1) find ProblemCategory by slug using the mapping, (2) if found, increment `usageCount` by 1, (3) create Assessment with status 'started'
- Return select projection: `{ id, problemType, status, createdAt }`
- Import ProblemType from `../generated/prisma/client`

**Function 2: `getAssessmentWithResponses`**
- Accepts: `id: string`
- Uses `prisma.assessment.findUnique` with `include` for responses ordered by `questionOrder: 'asc'`
- Select on assessment: `{ id, problemType, status, createdAt, responses: { id, questionId, answerText, answerValue, timeSpentMs, questionOrder } }`
- If not found, throw `NotFoundError('Assessment not found')`

**Function 3: `transitionStatus`**
- Accepts: `assessmentId: string, newStatus: AssessmentStatus`
- Define `VALID_TRANSITIONS` map: started->['in_progress'], in_progress->['completed'], completed->['report_generated'], report_generated->['unlocked']
- Find assessment by id (select status only). If not found, throw NotFoundError.
- If transition not in allowed list, throw `ValidationError('Cannot transition from X to Y')`
- Update and return the assessment with select: `{ id, status, updatedAt }`
- Import AssessmentStatus from `../generated/prisma/client`
