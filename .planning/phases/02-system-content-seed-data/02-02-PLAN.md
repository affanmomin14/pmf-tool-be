Create prisma/seed.ts that seeds all 5 system content tables using Prisma upserts.

CRITICAL DECISIONS:
- Import PrismaClient from `../src/generated/prisma/client` (NOT from src/db/prisma.ts which triggers env validation)
- Instantiate a fresh `new PrismaClient()` in the seed script
- Use upsert for idempotency: categories keyed on `slug`, questions on `displayOrder`, etc.

STRUCTURE:
```typescript
import { PrismaClient, QuestionType, ProofType } from '../src/generated/prisma/client';

const prisma = new PrismaClient();

async function seedCategories() { ... }
async function seedQuestions() { ... }
async function seedMicroInsights() { ... }
async function seedFacts() { ... }
async function seedSocialProof() { ... }

async function main() {
  console.log('Seeding system content...');
  await seedCategories();
  await seedQuestions();
  await seedMicroInsights();  // MUST run after seedQuestions (needs question IDs)
  await seedFacts();
  await seedSocialProof();
  console.log('Seeding complete.');
}

main()
  .then(() => prisma.$disconnect())
  .catch((e) => { console.error(e); prisma.$disconnect(); process.exit(1); });
```

SEED CATEGORIES (5): Use `prisma.problemCategory.upsert({ where: { slug }, update: { name, description, usageCount, displayOrder, isActive: true }, create: { name, slug, description, usageCount, displayOrder, isActive: true } })` for each. Exact data from FE CATEGORIES in seed_data_reference above.

SEED QUESTIONS (5): Upsert keyed on displayOrder (since there's no unique slug field on Question). Use a deleteMany + createMany approach instead -- delete existing questions and recreate. Actually, a better idempotent approach: upsert by using a combination. Since Question has no unique field besides id (autoincrement), use this pattern:
- For each question, use `prisma.question.upsert({ where: { id: N }, update: { questionText, questionType, placeholder, options, displayOrder, isActive: true }, create: { id: N, questionText, questionType, placeholder, options, displayOrder, isActive: true } })` with explicit IDs 1-5.
- QuestionType mapping: FE 'textarea' -> QuestionType.text, FE 'select' -> QuestionType.single_select
- Question 3 options: Store as JSON array: `[{ id: 'organic-search', label: 'Organic Search / SEO', value: 'organic-search' }, ...]` (all 8 options from FE)
- Include placeholder text for questions 1, 2, 4, 5. Question 3 has no placeholder (it's a select).

SEED MICRO-INSIGHTS (15): Must run AFTER questions are seeded. Since we used explicit question IDs 1-5, micro-insights can reference questionId directly.
- For each of the 15 insights, use `prisma.microInsight.upsert({ where: { id: N }, update: { ... }, create: { id: N, questionId, insightText, triggerKeywords: [], source: null, displayOrder, isActive: true } })` with explicit IDs 1-15.
- Question 1 insights: IDs 1-3, questionId 1
- Question 2 insights: IDs 4-6, questionId 2
- Question 3 insights: IDs 7-9, questionId 3
- Question 4 insights: IDs 10-12, questionId 4
- Question 5 insights: IDs 13-15, questionId 5

SEED FACTS (6): Use `prisma.pmfFact.upsert({ where: { id: N }, update: { ... }, create: { id: N, factText, source: null, displayLocation: 'loading', isActive: true } })` with explicit IDs 1-6.
- factText format: "${title}: ${description}" from FE PMF_FACTS. Example: "Only 1 in 4: startups achieve true product-market fit before Series A."

SEED SOCIAL PROOF (6): Use `prisma.socialProof.upsert({ where: { id: N }, update: { ... }, create: { id: N, quote, authorName, authorTitle: role, companyName, proofType: ProofType.testimonial, displayOrder: N, isActive: true } })` with explicit IDs 1-6.
- Extract companyName from role: "Co-founder, NoteStack" -> "NoteStack", "CEO, Briefcase" -> "Briefcase", etc.
- authorTitle is the full role string (e.g., "Co-founder, NoteStack")

IMPORTANT: Use the exact text from the FE constants. Do not paraphrase or modify any content.
