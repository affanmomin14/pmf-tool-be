Create an Express error-handling middleware (4 args: err, req, res, next). Logic:
1. If `err` is an instance of `AppError` (import from `../errors`), use its statusCode, code, message, and details.
2. If `err` is a Zod validation error (check `err.name === 'ZodError'` or similar), convert to ValidationError format: extract field-level issues into `details` array of `{ field: path.join('.'), message }`.
3. For all other errors: use 500 status, code "INTERNAL_ERROR", generic message.

5. In production: NEVER include stack trace.
6. Log the error using `logger.error({ err, statusCode }, 'Request error')`.
7. Response format: `{ success: false, error: { code, message, details?, stack? } }`.

**src/middlewares/validate.middleware.ts:**
Create a validation middleware factory. It accepts a Zod schema configuration object and returns Express middleware:

```typescript
import { z } from 'zod';
import { ValidationError } from '../errors';

type ValidationTarget = 'body' | 'params' | 'query';

interface ValidateOptions {
  body?: z.ZodType;
  params?: z.ZodType;
  query?: z.ZodType;
}

export const validate = (schemas: ValidateOptions) => {
  return (req, res, next) => {
    const errors: Array<{ field: string; message: string }> = [];

    for (const [target, schema] of Object.entries(schemas)) {
      const result = schema.safeParse(req[target as ValidationTarget]);
      if (!result.success) {
        for (const issue of result.error.issues) {
          errors.push({
            field: `${target}.${issue.path.join('.')}`,
            message: issue.message
          });
        }
      } else {
        // Replace req[target] with parsed (coerced/defaulted) value
        (req as any)[target] = result.data;
      }
    }

    if (errors.length > 0) {
      throw new ValidationError('Validation failed', errors);
    }
    next();
  };
};
```

**src/middlewares/requestLogger.middleware.ts:**
Import `pinoHttp` from `pino-http` and `logger` from `../config/logger`. Create and export middleware:

```typescript
import pinoHttp from 'pino-http';
import { logger } from '../config/logger';
import { randomUUID } from 'crypto';

export const requestLogger = pinoHttp({
  logger,
  genReqId: (req) => req.headers['x-request-id'] || randomUUID(),
  customLogLevel: (req, res, err) => {
    if (res.statusCode >= 500 || err) return 'error';
    if (res.statusCode >= 400) return 'warn';
    return 'info';
  },
  customSuccessMessage: (req, res) => `${req.method} ${req.url} ${res.statusCode}`,
  customErrorMessage: (req, res, err) => `${req.method} ${req.url} ${res.statusCode} - ${err.message}`,
  serializers: {
    req: (req) => ({
      id: req.id,
      method: req.method,
      url: req.url,
    }),
    res: (res) => ({
      statusCode: res.statusCode,
    }),
  },
});
```

The `genReqId` respects incoming `x-request-id` header (for distributed tracing) or generates a UUID. The request ID is accessible via `req.id` in downstream handlers.

```typescript
import express from 'express';
import cors from 'cors';

import { requestLogger } from './middlewares/requestLogger.middleware';
import { errorHandler } from './middlewares/error.middleware';

export const app = express();

// Security
app.use(helmet());
app.use(cors({
  origin: env.CORS_ORIGINS.split(',').map(s => s.trim()),
}));

// Body parsing
app.use(express.json({ limit: '1mb' }));

// Request logging (BEFORE routes so every request is logged)
app.use(requestLogger);

// Health check (no auth, no validation)
app.get('/health', (_, res) => {
  res.json({ success: true, data: { status: 'ok' } });
});

// === Future routes mount here ===

// Error handler (MUST be LAST middleware - after all routes)
app.use(errorHandler);
```

Key ordering:
1. helmet + cors (security, first)
2. express.json (parse body)
3. requestLogger (log every request)
4. Routes (health + future API routes)
5. errorHandler (catch-all, MUST be last)
