<task type="auto">
  <name>Task 1: Define complete Prisma schema with all models, enums, relations, and indexes</name>
  <files>prisma/schema.prisma</files>
  <action>
Replace the existing empty schema with the full database schema. Keep the existing generator and datasource blocks. Add a `url` to the datasource from `env("DATABASE_URL")`.

**Enums** (Prisma enums, NOT string unions):
- `AssessmentStatus`: started, in_progress, completed, report_generated, unlocked
- `PmfStage`: pre_pmf, approaching, early_pmf, strong
- `QuestionType`: text, single_select, multi_select, scale
- `ProblemType`: market_fit, product_quality, distribution, monetization, retention
- `ProofType`: testimonial, stat, case_study

**Domain 1 - Assessment Pipeline (UUID PKs via `@default(uuid())`)**:

`Assessment`:
- id: String @id @default(uuid())
- problemType: ProblemType
- status: AssessmentStatus @default(started)
- utmSource, utmMedium, utmCampaign: String? (nullable)
- ipHash: String? (for rate limiting, never raw IP)
- createdAt: DateTime @default(now())
- updatedAt: DateTime @updatedAt
- Relations: responses (Response[]), report (Report?), lead (Lead?)
- Index on status, index on ipHash, index on createdAt

`Response`:
- id: String @id @default(uuid())
- assessmentId: String (FK to Assessment, onDelete: Cascade)
- questionId: Int (FK to Question)
- answerText: String? (for text questions)
- answerValue: String? (for select questions, JSON-encoded if multi)
- timeSpentMs: Int? (time spent on question in milliseconds)
- questionOrder: Int (1-5, the display order position)
- createdAt: DateTime @default(now())
- Relations: assessment (Assessment), question (Question)
- Unique constraint on [assessmentId, questionId]

`Report`:
- id: String @id @default(uuid())
- assessmentId: String @unique (FK to Assessment, onDelete: Cascade)
- content: Json (the full 9-section report JSON)
- scores: Json (7-dimension score breakdown)
- previewContent: Json (2-3 insights for locked view)
- intermediateArtifacts: Json? (classification, research, scoring data)
- pdfUrl: String? (filled after PDF generation)
- pmfScore: Int (0-100)
- pmfStage: PmfStage
- primaryBreak: String (name of lowest dimension)
- aiModel: String (e.g. "gpt-4o")
- aiTokensUsed: Int
- aiCostCents: Int (cost in cents for precision)
- aiLatencyMs: Int
- urlToken: String @unique (nanoid for public access)
- expiresAt: DateTime (90 days from creation)
- createdAt: DateTime @default(now())
- updatedAt: DateTime @updatedAt
- Relations: assessment (Assessment)
- Index on urlToken, index on expiresAt

`Lead`:
- id: String @id @default(uuid())
- assessmentId: String @unique (FK to Assessment, onDelete: Cascade)
- email: String
- isUnlocked: Boolean @default(false)
- utmSource, utmMedium, utmCampaign: String? (separate from assessment UTM)
- createdAt: DateTime @default(now())
- updatedAt: DateTime @updatedAt
- Relations: assessment (Assessment)
- Index on email

**Domain 2 - System Config (Int auto-increment PKs)**:

`ProblemCategory`:
- id: Int @id @default(autoincrement())
- name: String @unique
- slug: String @unique
- description: String
- iconUrl: String?
- usageCount: Int @default(0)
- displayOrder: Int
- isActive: Boolean @default(true)
- createdAt: DateTime @default(now())
- updatedAt: DateTime @updatedAt

`Question`:
- id: Int @id @default(autoincrement())
- questionText: String
- questionType: QuestionType
- placeholder: String?
- options: Json? (array of option objects for select types)
- displayOrder: Int
- isActive: Boolean @default(true)
- createdAt: DateTime @default(now())
- updatedAt: DateTime @updatedAt
- Relations: responses (Response[]), microInsights (MicroInsight[])

`MicroInsight`:
- id: Int @id @default(autoincrement())
- questionId: Int (FK to Question)
- triggerKeywords: Json (array of keyword strings to match)
- insightText: String
- source: String? (attribution)
- displayOrder: Int
- isActive: Boolean @default(true)
- createdAt: DateTime @default(now())
- Relations: question (Question)

`PmfFact`:
- id: Int @id @default(autoincrement())
- factText: String
- source: String?
- displayLocation: String (e.g. "hero", "sidebar", "loading")
- isActive: Boolean @default(true)
- createdAt: DateTime @default(now())

`SocialProof`:
- id: Int @id @default(autoincrement())
- quote: String
- authorName: String
- authorTitle: String?
- companyName: String?
- proofType: ProofType
- displayOrder: Int
- isActive: Boolean @default(true)
- createdAt: DateTime @default(now())

**Domain 3 - Analytics/Audit (UUID PKs, createdAt only - append-only)**:

`AnalyticsEvent`:
- id: String @id @default(uuid())
- assessmentId: String? (nullable, some events are global)
- eventType: String (e.g. "assessment_started", "report_generated")
- eventData: Json?
- ipHash: String?
- userAgent: String?
- createdAt: DateTime @default(now())
- Index on eventType, index on assessmentId, index on createdAt

`ApiLog`:
- id: String @id @default(uuid())
- requestId: String (from pino request ID)
- method: String
- path: String
- statusCode: Int
- latencyMs: Int
- ipHash: String?
- createdAt: DateTime @default(now())
- Index on requestId, index on createdAt

`AiLog`:
- id: String @id @default(uuid())
- assessmentId: String? (FK nullable)
- provider: String @default("openai")
- model: String
- promptName: String (e.g. "classify", "generate_report", "research")
- inputTokens: Int
- outputTokens: Int
- totalTokens: Int
- costCents: Int
- latencyMs: Int
- success: Boolean
- errorMessage: String?
- createdAt: DateTime @default(now())
- Index on assessmentId, index on createdAt

`ResearchCache`:
- id: String @id @default(uuid())
- category: String
- subCategory: String
- data: Json (cached research results)
- expiresAt: DateTime
- createdAt: DateTime @default(now())
- Unique constraint on [category, subCategory]
- Index on expiresAt

**Important notes:**
- Use `@relation(fields: [...], references: [...], onDelete: Cascade)` on child side of all assessment-related FKs
- Response -> Question FK should NOT cascade (deleting questions shouldn't delete responses)
- Do NOT add `url` to the datasource `db` block -- it already reads from env by default via `prisma.config.ts`. Actually, Prisma 7+ with `prisma.config.ts` handles env loading, but the schema still needs `url = env("DATABASE_URL")` in the datasource block.

3. Verify the generated client exists and the migration SQL was created.

If the migration fails due to database connectivity (PostgreSQL not running), that is acceptable -- the schema validation in Task 1 already confirms correctness. In that case, just run `npx prisma generate` to produce the client types and note the migration needs to run when the DB is available.
